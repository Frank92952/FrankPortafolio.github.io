Informe de la Semana 14

En la semana 14, nos enfocamos en la instalación de un proyecto con Spring Framework y la configuración de una base de datos MySQL para crear una tabla en ella. A continuación, describiré los pasos principales para lograr esta configuración:

Se procesedio lo siguiente:

  • En primer lugar:Comenzamos creando un nuevo proyecto Spring utilizando Spring Initializr, una herramienta en línea que nos permite generar rápidamente la estructura básica del proyecto con las dependencias necesarias. Podemos seleccionar las dependencias requeridas, como Spring Web (para crear aplicaciones web), Spring Data JPA (para acceder a la base de datos) y MySQL (para trabajar con la base de datos MySQL).
  • A continuación: Asegurémonos de tener instalado MySQL en nuestra máquina o servidor. Podemos descargar MySQL desde el sitio oficial y seguir las instrucciones de instalación específicas para nuestro sistema operativo.
  • Durante la sesión: En el archivo de configuración de la aplicación Spring (application.properties o application.yml), agregamos la configuración necesaria para conectarnos a la base de datos MySQL. Configuramos el nombre de la base de datos, el nombre de usuario, la contraseña y otras propiedades de conexión.
  • Al final de la sesión: En Spring Data JPA, podemos definir una entidad que representa la tabla en nuestra base de datos. Una entidad es una clase Java que se mapea a una tabla en la base de datos. Podemos crear una clase Java, por ejemplo, Cliente, con anotaciones de JPA que indiquen cómo se debe mapear a la tabla en la base de datos.

Recuerda que Spring Data JPA simplifica en gran medida la interacción con la base de datos, ya que proporciona un mecanismo de abstracción para realizar operaciones CRUD (Crear, Leer, Actualizar, Eliminar) en la base de datos utilizando métodos de repositorio. Así, podemos centrarnos en el desarrollo de la lógica de negocio y dejar que Spring Data JPA se encargue de las operaciones de acceso a datos.
